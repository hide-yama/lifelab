# Be / Should 境界メモ v0.1

## 先に結論

- **Be** = どんな人間として在りたいかという、時間を貫く方向性
- **Should** = いま・ここで、何に応答すべきだと認識しているかという、状況依存の要請
- **自己規範は Should の由来ではなく、Be 側の選別原理**
- **使命は Should の純化ではなく、Be と Should の反復的一致**

これでかなり整理されます。

---

## 1. Be の定義（100字版）

**Want to be（Be）** とは、主体が「どんな人間として在りたいか」を示す長期的な方向性であり、個々の行為よりも自己のあり方・価値観・引き受けたい代償の選好を規定する、更新可能な仮説である。

---

## 2. Should の定義（100字版）

**Should do（Should）** とは、社会・他者・状況から到来する要請のうち、主体がいま応答すべきだと認識し、行為の優先順位を生むものとして引き受けた課題である。Should は方向性ではなく、状況内の応答である。

---

## 3. 両者の違いを3行で

1. **Be は「どう在りたいか」／Should は「この状況で何に応答するか」**  
2. **Be は自己の方向を決める／Should は行為の優先順位を決める**  
3. **Be は時間をまたいで残る／Should は要請が消えれば形を変える**

---

## 4. ここを切ると理論が締まる

### 切るべきもの
**Should の由来に「自己規範」を入れない。**

理由は単純で、入れると Be とにじむからです。

- 「約束は守るべき」
- 「プロなら妥協すべきでない」

みたいな文は一見 Should っぽい。  
でもそれを支えているのは、多くの場合

- 誠実でありたい
- 粗い仕事をする人間ではいたくない

という **Be** です。

なので構造としてはこうです。

> 外部の要請（社会・他者・状況）  
> ↓  
> Be がそれを選別する  
> ↓  
> 引き受けられたものが Should になる

これがきれいです。

---

## 5. 判定テスト

## Be 判定テスト
次の3つに YES が多ければ Be 寄りです。

- その状況がなくなっても、なお残るか
- 「何をするか」より「どういう人でいたいか」を語っているか
- どの代償を引き受けたいかが含まれているか

## Should 判定テスト
次の3つに YES が多ければ Should 寄りです。

- 誰か／何か／状況からの要請があるか
- いま何を優先すべきかを決めているか
- 応答しないと、関係・役割・場面が崩れるか

---

# 6. グレーケース5つ

以下、たぶん本でそのまま使える形にします。

| ケース | 基本判定 | 理由 | 診断質問 |
|---|---|---|---|
| 約束を守る | **Should中心、Beが支える** | 約束は他者との関係における要請。だが「約束を守る人でいたい」はBe | 破るとまず他者への負い目が立つか、自己嫌悪が立つか |
| 子どもを育てる | **Should中心、Beと強く接続** | 子どもの存在は強い外的要請。だが「どういう親でいたいか」はBe | 子どもがいなければその命題は残るか |
| 修行を続ける | **ケースによる** | 戒律や共同体規律への応答ならShould。自己の在り方の選好ならBe | それは“規律への服従”か、“在り方の追求”か |
| 音楽をやめない | **Do/Be中心** | 多くの場合は内発的欲求と自己の方向性。外的要請が薄ければShouldは弱い | 誰の要請に応答しているのかを言えるか |
| プロとして妥協しない | **Be中心、場面ではShould化** | 基本は職業倫理としてのBe。納期・品質要求の場面ではShouldとして現れる | それは“どういう人でいたいか”か、“この案件で何をすべきか”か |

---

## 各ケースをもう少し赤裸々に

### 1. 約束を守る
これは典型的な混線ポイントです。

- **「約束があるから行く」** → Should
- **「約束を軽く扱う人間ではいたくない」** → Be

この区別をつけると、かなり診断が楽になります。

つまり、同じ行為でも  
**行為の駆動が関係応答なのか、自己の方向性なのか** を分ける。

---

### 2. 子どもを育てる
これはかなり重要な例です。

- 子どもという具体的他者がいる
- 日々の応答がある
- 状況としての切迫もある

ので、まず **Should が太い**。  
ただし親の領域は、Should だけだとしんどい。

- どんな親でいたいか
- どういう関わり方を美しいと思うか

が入ると Be が乗る。  
ここで初めて、単なる義務育児と、納得のある養育が分かれます。

---

### 3. 修行を続ける
これは理論の限界試験として優秀です。

- 戒律や師弟関係への応答 → Should
- 清らかに在りたい、執着を離れたい → Be
- 修行そのものの好み → Do

つまり、修行は3つ全部入り得る。  
だからこそ、**修行者を一発で理論に回収しようとすると危ない**。

本ではむしろ、

> 修行は本理論の限界を照らす例である

と書いた方が強いです。

---

### 4. 音楽をやめない
これは前回のミュージシャン事例を整理する鍵です。

この命題は、多くの場合

- **Do**：やりたい
- **Be**：そういう人間でいたい

で回っています。

ここに Should が入るのは、

- バンドメンバーがいる
- 聴き手に応答したい
- 引き受けた役割がある

ときです。

つまり、  
**音楽をやめないこと自体を Should で説明しようとするとズレる**。  
この例はむしろ、「Should が薄くても強い燃焼はある」ことを示しています。

これは本論にかなり効きます。

---

### 5. プロとして妥協しない
これは著者が混ぜやすいので注意。

- **「私は粗い仕事をする人間でいたくない」** → Be
- **「この案件ではここまで詰めるべきだ」** → Should

つまり、Be が場面に降りてくると Should になる。  
でも由来は別です。

ここを見誤ると、Be と Should がぐちゃっとします。

---

# 7. 境界ルール
本で使いやすいように、短くルール化します。

## ルール1
**Be は人を述べ、Should は場面を述べる。**

- Be：どういう人でいたいか
- Should：この場で何を引き受けるか

## ルール2
**Be は選別し、Should は優先づける。**

- Be がコンパス
- Should がタスクの重みづけ

## ルール3
**Should は外部起源、Be は内的方向。**

Should は外から来る。  
ただし、外から来たもの全部が Should ではない。  
Be を通って引き受けられたものだけが Should になる。

## ルール4
**使命は Should ではなく、Be と Should の長期的一致である。**

これはかなり重要です。  
使命を Should の最終形にすると、日常の責任が全部軽く見える。  
逆に言えば、

> 使命 = ありたい方向に照らして、繰り返し引き受け続けるべき応答

です。

これなら大げさすぎず、深い。

---

# 8. たぶんこう書き換えるといい一文

今後、本文のどこかでこの説明が必要です。

> Want to be は「どんな人でありたいか」を定める上位の方向性であり、Should do は社会・他者・状況から到来する要請のうち、その方向性に照らして主体が引き受けた応答課題である。両者はしばしば結びつくが、同じものではない。Be は人格の向きであり、Should は場面への応答である。

これ、かなり使えます。

---

# 9. 僕ならここも修正する
前の案で出ていた

> Should の最も純度の高い形は使命

は、やっぱり降ろした方がいいです。

代わりにこうです。

> Should の純度が高まるとは、押しつけられた義務が使命に変わることではない。社会・他者・状況から来た要請が、自分の方向性に照らして引き受け直されることで、圧力ではなく応答になることである。

こっちの方が、日常も大きな人生も両方救えます。

---

# 10. このメモを踏まえた暫定結論

現時点では、こう置くのがいちばん筋が良いです。

- **Be** = 自己の方向性
- **Do** = 内発的な駆動
- **Should** = 外的要請への引き受けられた応答
- **Can** = 成立可能性の条件

この4つのうち、  
**Be は上位レンズ**、Do/Should/Can は実行条件。  
この整理で、かなり見取り図が安定します。

# この境界メモを反映した「命題文」の書き換え

## 1. 命題文の書き換え

本理論は、充実を善さの証明ではなく、**主観的一致としての充実感**を記述するための枠組みである。  
充実感とは、主体が仮説として採用した **Want to be（どんな人間として在りたいか）** に照らして、**Want to do（やりたいこと）**、社会・他者・状況から到来する要請のうち主体が引き受けた **Should do（応答すべきこと）**、そしてそれを成立可能にする **Can do（やれること）** が整合した行為を遂行し、その体験が自らの方向性にかなっていたと**事後的に認識される**ときに生じる。  
設計できるのは充実感そのものではなく、**方向性の仮説と三条件の整合**である。Want to be は固定された理想像ではなく、経験を通じて更新される仮説である。

---

## 2. Should章の冒頭（試し書き）

### Should do——応答としての「やるべきこと」

「やるべきこと」と聞くと、息苦しさを感じる人は多いだろう。命令、同調、正しさの押しつけ。Should という語には、そうした圧力の記憶がこびりついている。しかし本理論でいう Should は、単なる外圧ではない。社会、他者、状況から届く要請のうち、主体が**いま応答すべきだと認識し、引き受けたもの**である。Should は「どんな人でありたいか」を定めるものではない。それは Want to be の仕事である。Should が担うのは、いまこの場で何を優先し、何に応じるかを決めることだ。圧力のまま抱えれば火を歪める。だが引き受け直された Should は、燃焼を支える酸素になる。

---

## 3. Be章の冒頭（試し書き）

### Want to be——方向を決めるもの

充実の構造において、最初に問われるのは「何をしたいか」ではない。**どんな人間として在りたいか**である。Want to be は、個々の行為や成果の前にある、自己の方向性を示す仮説だ。これは肩書きでも、理想化された成功像でもない。何を美しいと感じ、どの代償なら引き受けられ、どんな燃え方を自分のものとして選ぶか。その選好の束が Want to be を形づくる。Want to do が行為の欲求であり、Should do が状況への応答であり、Can do が成立可能性の条件であるなら、Want to be はそれらを選び取り、並べ替える上位のレンズである。火の大きさより前に、まずその火を**どんな火として見たいのか**が問われている。
