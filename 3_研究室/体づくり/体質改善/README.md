## 改善ロードマップ（優先順）

1. 夜のコーヒーの影響を検証する（廃止ありきではなく実験として）

6. ストレッチの習慣化
7. 早起きスケジュールとの連動

## 毎日の問診（PDCAシステム）

毎朝のセッション冒頭でAIトレーナーが問診を行い、体調の変化を追跡する。
データとアドバイスをセットで記録し、経過観察として振り返れる形にする。

### 朝の問診（体の状態）

1. **就寝→起床**（時刻）
2. **睡眠の質**（熟睡できた / まあまあ / 浅かった）
3. **回復感**（1-5）: 1=ぐったり / 2=いまいち / 3=普通 / 4=まあまあ / 5=スッキリ
4. **体の状態**（自由記述）: 肩首こり、目の疲れ、背中、胃腸 など
5. **夜コーヒー**（時間帯 or 飲まなかった）
6. **飲酒**（種類・量 or 飲まなかった）
7. **エネルギーレベル**（高い / やや高め / 普通 / 低め）
8. **気分・メンタル**（一言）
9. **介入変数の実施**（やった/やらなかった）
10. **Phase 1 習慣の追跡**（※現フェーズに応じて項目を更新する）
    - 朝散歩（した / しなかった）
    - 朝のショート動画（見てない / 10分以内 / 30分程度 / 1時間以上）
11. **会話**: 選択式の後に「昨日と比べて変わったことや気になることは？」と聞く。本人の言語化から気づきを拾い、アドバイスに繋げる

### 夕方の問診（脳のパフォーマンス）

1. **頭のクリアさ**（1-5）: 1=ぼんやり / 2=重い / 3=普通 / 4=まあまあ / 5=シャープ
2. **集中力の持続**（1-5）: 1=すぐ切れた / 2=短かった / 3=普通 / 4=長め / 5=没頭できた
3. **午後の眠気**（なし / 軽い / 強い）
4. **会話**: 選択式の後に「今日1日やってみてどうでしたか？」と聞く。体感の変化やパフォーマンスの気づきを拾う

### 問診の進め方

- **形式**: AskUserQuestion（ターミナル上の選択式UI）を使う。テキストで選択肢を並べるのではなく、必ず選択UIで出す
- **最初に聞く**: 「がっつり問診しますか？ さくっと済ませますか？」
  - **がっつり**: 基本項目＋深掘り（飲酒量、ショート動画、朝の過ごし方、集中力、エネルギー、モチベーション、睡眠の質など）＋1日のアドバイス
  - **さくっと**: 基本項目（就寝起床・回復感・体の状態・夜コーヒー・気分・介入実施）のみ
- **会話パート**: 選択式の後に自由記述で補足してもらう

### ルール

- **介入**: 1変数ずつ、最低2週間ずつ
- **振り返り**: 週末にデータを見て傾向を確認
- **記録**: 月別ファイルに蓄積（`問診ログ/YYYY-MM.md`）
