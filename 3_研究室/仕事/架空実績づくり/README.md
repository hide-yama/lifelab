# 架空実績づくり

**ステータス**: 構想中

架空だがリアルなクライアントワークをシミュレーションし、営業やセルフブランディングに使えるレベルの成果物を作る。

## なぜやるのか

- 実績がないから受注できない→受注できないから実績ができない、のループを突破する
- 「御社の課題に近いケースでこれだけやれます」と見せられる状態を作る
- 自分のスキルの棚卸しと、提供価値の言語化にもなる
- 事業家としての第一歩。クライアントワークの延長ではなく、自分の事業の種を見つける場でもある

## 自分の強み

### 経歴
- アクセンチュア出身。コカ・コーラのコークオンのアプリ開発・キャンペーン運用に関わる
- フロントエンドエンジニア → Webデザイナー → ディレクター（現在）
- 個人事業主として独立

### コアバリュー: 「現場がわかるから上流もできる」
- 戦略（コンサル・マーケ）から実装（デザイン・コーディング）まで全レイヤーを自分で経験している
- 現場がわかるから、実現可能な戦略が立てられる
- コストの肌感覚がある → リソース配分が的確
- チームビルディングとディレクションの現場力が強み
- AIを使ったアプリ開発を実践中 → 少人数でも高速にプロダクトを作れる

### 見せたいのは
「何でも屋」ではなく、**経営課題を理解して、戦略を立てて、実行まで最短距離で持っていける人**。大企業が雇うコンサル＋制作会社＋マーケ会社を一人（または少人数）で兼ねられる存在。

## ゴール: 事業家への道

最終的にはクライアントワークで終わらない。自分が事業をやるときにも、現場がわかるからこそコストを低くしてリソースを有効活用しながら、自分をクライアントとして成功に持っていく。架空実績づくりはその過渡期の武器。

## 提案: こんな進め方はどうか

### Step 1: 架空クライアントを設定する
中小企業をターゲットに、リアルな会社像を作る。

ソリューションの幅は広い。案件によって武器を使い分ける：
- **業務効率化アプリ** — SaaSのオーバースペック問題を解く。ピンポイントで効くカスタムメイドアプリ。大企業のSIerが入らない規模感
- **マーケティング戦略** — 集客・ブランディング・キャンペーン設計
- **DX推進** — 紙やエクセルからの脱却、業務フローの再設計
- **上記の組み合わせ** — 戦略＋実装をセットで提供するのが本当の強み

### Step 2: 課題分析レポートを作る
AIをリサーチャーとして使い、その業界・業種のリアルな課題を徹底的に調べる。
- 業界の構造的な課題
- 同規模の企業がよくぶつかる壁
- 競合の動向

→ 「この人、うちの業界のことわかってるな」と思わせるレベルまで掘り下げる。

### Step 3: ソリューションを設計する
課題に対して、最適なアプローチを設計する。
- 戦略の全体像（何をどの順番でやるか）
- 必要なリソースとコスト感
- 自分が担う範囲とチームが必要な範囲

### Step 4: 実際に作る
設計だけでなく、成果物を実際に制作する。
- アプリならプロトタイプを開発（AIを使って高速に）
- マーケなら戦略資料＋施策の具体案
- 動くもの・見せられるものがあることが最大の差別化

→ 「提案書」ではなく「もう動いています」。

### Step 5: ケーススタディとしてまとめる
一連の流れを1つのストーリーとしてまとめる。
- 課題ヒアリング → 分析 → 戦略 → 実行 → 想定される成果
- 「御社の課題に近いケースで、ここまでやりました。御社向けにカスタマイズしてみませんか？」と言える状態に

## まず最初の1本をどうするか

1. 架空クライアント候補を3つ考える（業種×課題の組み合わせ）
2. その中から1つ選ぶ（自分が興味を持てる＋営業先として現実的なもの）
3. 課題分析から始める

全工程をAIと一緒に進める。リサーチはAI、設計と制作は自分、壁打ちと整理はAIという役割分担。
