# デイリールーム

1日1ファイル。タスク選び・実行記録・振り返りを1箇所にまとめる場所。日誌も兼ねる。

## ファイル名

`YYYY-MM-DD.md`（例: 2026-03-11.md）

## セクション構成

### 1. 今日の研究テーマ
- 朝の作戦会議で1つ決める
- 研究室のテーマから選ぶ（体づくり、デジタルデトックス、こころ、お金、住まい）
- 翌日に持ち越してもいい。気分転換に別テーマに切り替えてもいい
- ただし同時に複数は走らせない

### 2. 研究メモ
- テーマに取り組んだ内容を書く
- 研究室ごとにサブ見出し（`###`）で分ける
- 研究テーマ以外の活動も「その他」として記録してよい

### 3. タイムライン
- その日の時間の使い方を記録する
- 書式: `HH:MM 内容` をシンプルに並べる（箇条書きなし）

### 4. 今日のタスク（Todoistより）
- Todoistから取得したタスクを優先度別に並べる
- 前日から持ち越しがある場合は「p1（昨日からの持ち越し）」のように明記する
- セクションの区分:
  - `p1（今日やる）` — 今日やると決めたもの
  - `p1（朝ルーティン）` — ルーティン系のp1があれば分ける
  - `p2（重要だが急がない）`
  - `p3（緊急だがパッと片付ける系）`
  - `p4（いつでもいい）`
  - `完了済み` — 完了したタスクをここに移動する

### 5. 振り返り
- 1日の終わりに書く
- 何ができたか、何ができなかったか、気づきなど

## ルール

- **タスク名はTodoistからそのまま転記する** — AIが勝手に要約・短縮してはいけない。一字一句そのままコピーすること
- **タスク管理の正はTodoist** — デイリーファイルはその日のスナップショット。タスクの追加・完了処理はTodoistで行う
- **1日1研究テーマ** — 「際限なくPCの前にいてしまう」パターンを防ぐためのルール。終わったら今日は終わり
